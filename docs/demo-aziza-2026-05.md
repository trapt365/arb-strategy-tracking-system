## 0. Pre-flight чеклист (Тимур, до звонка с Азизой)

### 0.1 Настройка .env

Открыть `C:\Users\Timur\projects\strategy-tracking-system\.env` и убедиться что есть все переменные:

```
# Исправить имя ключа (сейчас API_KEY_CLAUDE → должно быть):
ANTHROPIC_API_KEY=sk-ant-...

# Добавить (если отсутствуют):
TELEGRAM_BOT_TOKEN=<токен бота из @BotFather>
TELEGRAM_CHAT_WORK_ID=<chat ID рабочей группы или личного чата Тимура>
TELEGRAM_CHAT_OPS_ID=<chat ID для ops-алертов — можно тот же что work>
TELEGRAM_TRACKER_CHAT_IDS=<chat ID Азизы в Telegram>

GEONLINE_F0_SHEET_ID=<ID Google Sheet с OKR + stakeholders>
GOOGLE_SERVICE_ACCOUNT_JSON=./data/google-service-account.json

# Уже есть:
SONIOX_API_KEY=...
```

> **Как узнать chat ID Азизы:** попросить её написать @userinfobot в Telegram — он ответит её ID.

> **Как получить TELEGRAM_BOT_TOKEN:** если бот уже создан в @BotFather — скопировать токен оттуда. Если нет — создать через /newbot.

### 0.2 Запуск Docker (из Windows Terminal, не WSL)

```cmd
cd C:\Users\Timur\projects\strategy-tracking-system
docker compose up -d
```

Проверить что бот поднялся (через 15-20 сек):

```cmd
docker compose ps
docker compose logs --tail 20
```

Ожидаемый лог: `Bot started` / `/health 200 OK`.

### 0.3 Проверить бот лично

Открыть бот в Telegram (по имени из @BotFather), написать `/start` — должен прийти ответ с инструкцией.

Если бот молчит — смотреть логи: `docker compose logs --tail 50`.

### 0.4 Иметь ссылку на запись встречи

Для demo нужна любая реальная запись встречи Geonline (Google Drive или Zoom URL). Лучше — недавняя, с которой Азиза знакома.

## 1. Звонок с Азизой (~15 мин)

### 1.1 Онбординг (3-5 мин)

**Тимур:**
> «Я добавил тебя в бот, который будет помогать готовить отчёты по встречам. Сейчас покажу как это работает.»

**Азиза:** находит бот в Telegram (имя бота скажет Тимур) → пишет `/start`.

Бот ответит приветствием. Азиза читает. Если вопросы — `/help`.

### 1.2 Первый отчёт вместе (~10 мин)

**Тимур:**
> «Возьмём ссылку на запись встречи [конкретная встреча]. Ты пишешь боту: /report [ссылка]»

**Азиза:** пишет `/report <ссылка на запись>`.

**Что происходит:**
- Бот немедленно отвечает: «Принял. Обрабатываю...» + прогресс-сообщения
- Обработка занимает 10–15 минут (транскрипция + анализ Claude)
- Бот присылает preview отчёта с тремя кнопками

**Кнопки:**
- `✅ Подтвердить → [Имя топа]` — отчёт готов, бот вернёт текст для WhatsApp
- `✏️ Исправить` — Азиза отвечает reply с правками → бот пересоздаёт preview
- `❌ Отклонить` — перегенерация или ручной режим

**После approve:**
Бот вернёт готовый текст — скопировать и вставить в WhatsApp/мессенджер топу.

### 1.3 Что показать Азизе отдельно

- Как выглядит сообщение «бот занят» если запрос уже в процессе
- Как написать правку через reply (пример: «добавь пункт про бюджет»)
- Что делать если бот не отвечает: написать Тимуру

## 2. Самостоятельная работа Азизы

### Стандартный процесс (после каждой встречи)

1. Открыть Telegram → найти бот
2. `/report <ссылка на запись Google Drive или Zoom>`
3. Подождать 10–15 минут
4. Проверить preview → `✅ Подтвердить` (или исправить)
5. Скопировать готовый текст → отправить топу в WhatsApp

### График доступности бота

Бот работает **только когда запущен Docker на ноутбуке Тимура**.
Стандартно: бот доступен в рабочие дни по расписанию встреч (пн/чт).

Если бот не отвечает → написать Тимуру.

---

## 3. Ограничения пилота (Азиза должна знать)

| Ограничение | Что делать |
|-------------|-----------|
| Бот недоступен (Docker выключен) | Написать Тимуру → он запустит |
| Бот «завис» / нет ответа >20 мин | Написать `/report <ссылка>` повторно |
| Отчёт некорректный | `✏️ Исправить` reply с конкретными правками |
| Нужен отчёт срочно | Позвонить Тимуру — он запустит бот |
