## Что делает система

AI-ассистент слушает запись встречи, пишет структурированный отчёт, ты проверяешь и отправляешь клиенту. Экономит ~2 часа на каждую встречу, фиксирует обязательства с цитатами (accountability).

---

## Быстрый старт (3 шага)

1. Получи аудиозапись встречи (Zoom Cloud / Google Drive share link).
2. Отправь боту: `/report <ссылка>`.
3. Через 3-5 минут получи черновик отчёта → проверь → нажми **Approve** или **Edit**.

---

## Чеклист подключения нового клиента

Перед первой встречей через pipeline **обязательно**:

- [ ] **Согласие всех топов подписано** (см. [consent-template-v1.0.md](consent-template-v1.0.md))
  - Без подписи конкретного топа — не записываем встречи с его участием.
  - В случае отказа — ручные заметки без AI.
- [ ] **F0 данные собраны** (Google Sheets шаблон):
  - `_okr` — цели и KR клиента на квартал.
  - `_stakeholder_map` — карта ролей: кто за что отвечает.
  - `_metrics` — 2 ключевые метрики (leading + lagging) по каждому департаменту.
  - Консистентность заголовков: snake_case, без пробелов.
- [ ] **Конфиг клиента создан** (делает Тимур): `clients/{slug}.yaml` — ссылка на Sheets, chat_id, тайм-зона.
- [ ] **Тестовая встреча проведена:** первая запись → ручная сверка отчёта с транскриптом → обсуждение с клиентом.

## Обычный цикл работы

### Шаг 1. `/report <ссылка>`

Отправь боту в рабочем чате Telegram:

```
/report https://drive.google.com/file/d/XXX/view?usp=sharing
```

Или ссылку на Zoom Cloud Recording:

```
/report https://zoom.us/rec/share/XXX
```

**Важно:** ссылка должна быть доступна на чтение (проверь sharing permissions).

### Шаг 2. Ожидание pipeline (3-5 мин)

Бот пришлёт прогресс-обновления:

- ⏳ **Скачиваю аудио...** (~30 сек)
- 🎙 **Транскрибирую (Soniox)...** (1-3 мин, зависит от длительности встречи)
- 🧠 **Анализирую (Claude)...** (1-2 мин)
- ✅ **Черновик готов** → присылает текст отчёта + inline-кнопки.

### Шаг 3. Review отчёта

Бот присылает отчёт в виде сообщения + кнопки:

- **✅ Approve** — отчёт ок, отправить клиенту.
- **✏ Edit** — нужны правки → бот присылает текст в редактируемом формате.
- **❌ Reject** — отчёт непригоден, не отправлять.

### Шаг 4. Edit flow (если нужны правки)

1. Нажми **✏ Edit** → бот пришлёт текст отчёта.
2. Отредактируй прямо в сообщении (Telegram edit), либо отправь новую версию боту.
3. Отправь команду `/approve_edited` или нажми Approve повторно.
4. Бот сохранит отредактированную версию и доставит клиенту.

### Шаг 5. Delivery

После **Approve** бот:

- Сохраняет отчёт в Google Sheets (лист `_reports` или отдельный).
- Отправляет клиенту по настроенному каналу (Telegram клиента или email — зависит от конфига).
- Присылает тебе подтверждение: «Отчёт доставлен клиенту Y».

## Fallback при сбоях

### Soniox не смог транскрибировать

**Признак:** бот пишет «Ошибка транскрипции» или pipeline застрял >10 мин.

**Действия:**
1. Проверь ссылку на аудио — доступна ли на чтение.
2. `/report <ссылка>` повторно через 5 мин.
3. Если 2 попытки неудачны → эскалировать Тимуру (@timur).
4. Альтернатива: скачать `.txt` транскрипт вручную (любой провайдер) → отправить файлом в бот через `/upload_transcript` (реализация в Story 1.2).

### Claude API timeout

**Признак:** этап «Анализирую» занимает >5 мин.

**Действия:**
1. Подождать ещё 2 мин (иногда API-очередь медленная).
2. `/status` — если pipeline всё ещё работает, не отменять.
3. Если >10 мин без прогресса → эскалировать Тимуру.

### Полный сбой бота

**Признак:** бот не отвечает 5+ минут, `/status` не работает.

**Действия:**
1. Написать в ops-канал: «бот не отвечает».
2. Если ops-канал тоже молчит → Telegram Тимуру + Айдару.
3. Ручной режим: открыть транскрипт, составить отчёт по шаблону `docs/manual-report-template.md` (TODO: создать в Story 1.5).

## Правило чувствительности контента

**ДО нажатия Approve** проверь отчёт на чувствительные моменты.

### Red-flag категории

| Категория | Пример | Что делать |
|-----------|--------|------------|
| Личные конфликты | «Иван сказал что Петров некомпетентен» | Обобщить: «в команде есть вопросы к ролям» |
| Зарплаты / деньги | «CFO упомянул зарплату Маркетинга 2.5М» | Удалить или обобщить до диапазона |
| Коммерческая тайна | Конкретные контракты, суммы сделок | Удалить, если клиент не одобрил явно |
| Третьи лица без согласия | Упоминание сотрудника, который не на встрече, с оценкой | Удалить ФИО, оставить роль |
| Здоровье / личная жизнь | «X разводится, поэтому падает продуктивность» | Удалить полностью |

### Правило

> **Если сомневаешься — обобщи или удали.** Лучше убрать точность, чем нарушить доверие клиента или чьё-то право на приватность.

### Примеры

**Плохо:**
> «Иван Петров (CFO) сказал, что Мария Сидорова из HR некомпетентна в вопросах найма и её нужно заменить.»

**Хорошо:**
> «CFO поднял вопрос компетенций в HR-функции и предложил пересмотреть подход к найму.»

## Когда эскалировать

- **Ошибка pipeline 2+ раза подряд** → Тимур.
- **Клиент жалуется на отчёт** (фактическая ошибка, искажение цитаты) → Тимур + Айдар. Сохрани транскрипт + отчёт для разбора.
- **Требование удалить данные от топа** (отзыв согласия) → Тимур (удаление за 30 дней по 94-V).
- **Подозрение на утечку** (бот прислал отчёт не тому клиенту) → немедленно Тимур + Айдар.
