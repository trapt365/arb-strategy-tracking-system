# Вопросник онбординга клиента v1.0

Дата: 2026-07-09 · Синтезирован из материалов Vault практики (источники — в конце) по решению дизайн-записки design-onboarding-unified-entry-2026-07-09.md (§7.1).
Назначение: контент для бота — **Часть A** питает шаг «Профиль клиента» (story 9.1), **Часть B** — путь «Вопросник» (story 9.5). Ответы — текстом, голосом 🎤 (до 5 мин) или документом 📎.
Цели: качественно описать клиента, текущее состояние, ожидания — и провести качественную проблематизацию.

## Принципы (из методологии практики)

1. **Вопрос → верифицируемая гипотеза.** Каждый вопрос существует, чтобы подтвердить/опровергнуть гипотезу о клиенте или наполнить конкретное поле артефакта. Вопросы «для интереса» вычеркнуты (матрица покрытия — дизайн-доки SAM).
2. **Grounding-тест:** хороший ответ указывает на наблюдаемое изменение в реальном мире («что должно измениться, когда мы это сделаем»), а не на мнение.
3. **Факт vs оценка.** Числа помечаются: точная цифра или прикидка. Расхождение «восприятие vs число» — само по себе диагноз (пары SAM v2).
4. **Каждый открытый вопрос — с примером ответа.** Пример калибрует детализацию и снимает «не знаю, как отвечать» (стандарт SAM).
5. **«Не знаю» — тоже данные.** Ответ пропускается без борьбы → поле уходит в 🔴 и в дозаполнение; незнание фиксируется как пробел.
6. **Не подсказывать ответ.** Формулировки без вшитых допущений (не «что мешает при быстром росте», а «что мешает»); на «так делают все» — встречный вопрос «а чем выделитесь вы?» (Verum).

## Два уровня: критический минимум 🔑 и расширенный опросник

- **🔑 Критический минимум** — вопросы, без ответов на которые таблица стратегического трекинга не собирается: название компании (заголовки), суть бизнеса (grounding LLM), топы с зонами ответственности (персональные листы, владельцы KR), направления, KR «с X до Y к сроку», ответственные. Бот задаёт их обязательно и в первую очередь.
- **Остальные вопросы (без 🔑) — расширенный опросник, опционально.** После критического минимума бот предлагает: «Минимум для таблицы собран. ➕ Заполнить расширенный профиль (лучше контекст отчётов и гипотез) или продолжить?» Расширенную часть можно заполнить и позже, в любой момент — ответы дописываются в карточку клиента и контекст.
- Пропуск расширенного вопроса ничего не блокирует; пропуск 🔑-вопроса → штатное 🔴-дозаполнение до `/confirm`.

| 🔑 A1.2 | Чем занимается компания и для кого? Отрасль, продукт, кто клиент. | открытый. Пример: «Образовательный центр подготовки к ЕНТ, Алматы+онлайн; клиенты — родители школьников 9–11 классов» |

Пример: «Дамир — коммерческий директор, полный P&L продаж, зона: выручка и воронка» → субъекты трекинга, персональные листы Sheets |

### A4. Запрос и ожидания (проблематизация первого прохода)

| # | Вопрос | Формат / пример |
|---|---|---|

| A4.2 | Почему это стало актуальным именно сейчас? Что случилось? | открытый — триггер отделяет реальную боль от «давно пора» |
| A4.3 | Что уже пробовали, чтобы решить (консультанты, школы, свои попытки)? Что не сработало и почему? | открытый. Пример: «Прошли курс Х, заказывали маркетинг-стратегию — не были готовы внедрять» |
| A4.4 | Образ результата: что будет успехом через 3 месяца? А через год? | два открытых. Пример (год): «Кассовый разрыв ≤500 млн, EBITDA 12%, я не в операционке» |
| A4.5 | Желаемые финансовые показатели — одна амбициозная цифра с горизонтом | число+срок. Пример: «×2 по выручке за учебный год» — практика всегда доводит хотелку до одной цифры |
| A4.6 | Главный приоритет — расставь 1-2-3: скорость роста / максимальная прибыль / минимизация рисков / выход собственника из операционки / доля рынка | ранжирование (кнопки) — вопрос-якорь Verum: «от ответа зависит вся структура проекта» |

**Выход части A:** заполненная карточка клиента (ClientCard) = grounding-контекст для всех путей онбординга и всех артефактов.

## Часть B. Вопросник стратегии (story 9.5 — путь «Вопросник»)

### B1. Видение и цели года

| # | Вопрос | Формат / пример |
|---|---|---|
| B1.1 | Какой ты видишь компанию через 3 года? 2–3 характеристики с числами | открытый. Пример: «150+ человек, 200+ млрд ₸, лидер ниши в Казахстане» |

| 🔑 B1.3 | Назови 3–5 направлений (целей года), по которым пойдёт стратегия | по одному, «✅ готово» кнопкой → objectives |

### B2. Ключевые результаты по направлениям (цикл по каждому направлению из B1.3)

| # | Вопрос | Формат / пример |
|---|---|---|
| 🔑 B2.1 | Как поймём, что направление «{X}» получилось? Метрика «с X до Y к сроку» | число+число+срок (валидация 8.6). Grounding-тест: что изменится в реальном мире |
| 🔑 B2.2 | Кто отвечает за этот результат? | **кнопки из топов профиля A3.2** — свободный ввод имён закрыт |

### B3. Текущее состояние (симптомы)

| # | Вопрос | Формат / пример |
|---|---|---|
| B3.1 | Главное ограничение роста — выбери до 2: деньги / люди / процессы / управленческая система / рынок и спрос / отсутствие ясной стратегии / другое | чекбоксы + «другое» |
| B3.2 | Какой % времени ключевых людей уходит на стратегические задачи (не операционку)? | число-прикидка |
| B3.3 | Какие 2–3 показателя вы уже отслеживаете регулярно? | открытый; пустой ответ = нет культуры показателей — это данные |
| B3.4 | Что происходило с прошлыми стратегическими задачами: какой % выполнялся в срок, возвращались ли к ним? | открытый — прямая диагностика потребности в трекинге |

### B4. Проблематизация (корневые причины)

| # | Вопрос | Формат / пример |
|---|---|---|
| B4.1 | Насколько тебя тревожит масштаб задуманного (1–10)? Что именно вызывает беспокойство? | шкала + открытый — эмоция отдельно от когниции (триада SAM) |
| B4.2 | Насколько понимаешь МЕХАНИЗМ достижения цели (1 = «верю, но не знаю как», 10 = «вижу каждый шаг»)? | шкала — разрыв B1.2 против B4.2 = главный материал сессии |
| B4.3 | Тест узкого места: если ограничения из B3.1 станет вдвое больше (людей/денег/места) — вырастете вдвое? Если нет — что упрётся следующим? | открытый — отделяет заявленный bottleneck от реального |
| B4.4 | Вспомни изменение, которое не взлетело: на каком этапе остановилось и что мешало лично тебе его поддерживать? | открытый. Пример: «Неуверенность в команде внедрения, не было быстрых побед» (паттерн Садыхан) |
| B4.5 | Разложи причины текущей ситуации: внешние факторы / внутренние / что можно было сделать иначе | три коротких открытых — связка ломает «поставщики подвели» (урок анализа Geonline) |

### B5. Гипотезы роста

| # | Вопрос | Формат / пример |
|---|---|---|
| B5.1 | Сформулируй 2–3 гипотезы по шаблону «ЕСЛИ (действие) — ТО (эффект с числом) — ПОТОМУ ЧТО (основание)» | по одной. Пример: «ЕСЛИ введём рассрочку на годовые пакеты — ТО конверсия вырастет с 20% до 26% — ПОТОМУ ЧТО 86% и так платят в рассрочку»; без шаблона приходят «тактические действия, а не гипотезы» |
| B5.2 | По каждой: чем проверим? Какая метрика изменится и когда посмотрим? | метрика+срок → банк гипотез (инвариант 2: гипотеза без метрики → 🔴) |

### B6. Финал

| # | Вопрос | Формат / пример |
|---|---|---|
| B6.1 | Что может помешать этой работе? (страх открытости / доминирование отдельных людей / нет времени / «много разговоров, мало решений» / другое) | чекбоксы |
| B6.2 | Какие темы ОБЯЗАТЕЛЬНО должны быть в трекинге, даже если мы про них не спросили? | открытый — обязательный финал каждого опросника практики |

## Правила ведения диалога (для реализации в боте)

- **Сначала 🔑-минимум, потом предложение расширенного.** Бот задаёт критические вопросы обязательно; после них — кнопка «➕ Расширенный опросник» / «Дальше». Расширенный доступен и позже (из карточки клиента), ответы дописываются в профиль/контекст.
- По одному вопросу; заголовок блока при входе в блок (паттерн 8.6). Прогресс `(i/N)` — отдельно для минимума и расширенной части.
- В каждом открытом вопросе — пример ответа и подсказка «🎤 голосом / 📎 документом / /skip».
- Числовые ответы — мягкая валидация 8.6 (один переспрос, повтор принимается).
- Пропуск/«не знаю» не блокирует: поле → 🔴 → штатное дозаполнение.
- Ответственные (B2.2) и decision maker (A3.3) — только кнопками из профиля.
- Persist после каждого ответа (механика 7.3) — вопросник переживает рестарт.
- Порядок блоков — драматургия SAM: тяжёлые открытые (A4, B1) ближе к началу, кнопочные (B3, B6) — как передышка; ожидания не в самом конце.
