## Инструкции

1. **OKR-покрытие (okr_coverage):** Для КАЖДОГО KR из OKR-контекста определи статус обсуждения:
   - `"discussed"` — KR обсуждался содержательно (>= 2 реплики по существу)
   - `"mentioned"` — KR упомянут, но без содержательного обсуждения
   - `"blind_zone"` — KR вообще не затронут на встрече

   Для каждого KR укажи:
   - **kr** — номер/название KR
   - **status** — один из трёх статусов выше
   - **mentions_count** — количество содержательных упоминаний
   - **substance** — true если >= 2 содержательных реплик

2. **Статус гипотез (hypothesis_status):** Если на встрече обсуждались гипотезы или эксперименты, определи их статус:
   - `"idea"` — только идея, обсуждение
   - `"in_test"` — тестируется, есть план проверки
   - `"result"` — есть результат (данные, метрики, выводы)

3. **Алерты (alerts):** Зафиксируй аномалии:
   - KR не обсуждался 2+ недели подряд (blind zone)
   - Противоречия между заявлениями и фактами
   - Обязательства без сроков или ответственных
   - Спикеры из `speaker_check` (из Step 1)
   - Незакрытые/просроченные обязательства из прошлых встреч (если deadline истёк, а статус не `completed`)
   - Цитаты с `approximate: true` — пометь как "цитата с приближённой меткой времени"

4. **Статусы commitments из прошлых встреч (commitments_status_updates):** Если массив открытых обязательств выше непуст — для каждого определи актуальный статус на основе обсуждений в текущей встрече:
   - `"open"` — без изменений (обязательство в работе, deadline ещё не истёк)
   - `"completed"` — явно выполнено (есть подтверждение в транскрипте)
   - `"overdue"` — deadline истёк, ничего не сделано

   Для каждого обновления верни объект с полями `who`, `what`, `previous_quote` (цитата из старого commitment), `new_status`, `evidence_quote` (цитата из текущей встречи, опционально). Если открытых обязательств нет (пустой массив `[]`) — пропусти этот раздел и верни пустой массив.

## Формат ответа

Верни ТОЛЬКО валидный JSON (без markdown fences):

{
  "okr_coverage": [
    {
      "kr": "KR номер или название",
      "status": "discussed",
      "mentions_count": 3,
      "substance": true
    }
  ],
  "hypothesis_status": [
    {
      "hypothesis": "Название гипотезы",
      "status": "in_test",
      "evidence": ["Факт 1", "Факт 2"]
    }
  ],
  "alerts": ["Алерт 1", "Алерт 2"],
  "commitments_status_updates": [
    {
      "who": "Жанель",
      "what": "подготовить отчёт",
      "previous_quote": "[01:23] ...",
      "new_status": "completed",
      "evidence_quote": "[15:44] ..."
    }
  ]
}

## Критически важно

- Статус `"discussed"` только при >= 2 содержательных реплик. Одно упоминание = `"mentioned"`.
- `"blind_zone"` — объективный факт отсутствия обсуждения, НЕ оценка.
- Алерты должны быть конкретными и actionable, не общими фразами.
- `commitments_status_updates` — пустой массив, если открытых обязательств нет.
