## Инструкции

Сформатируй отчёт **максимум в 3 секции** (если алертов нет — 2 секции):

### Секция 1: Ключевые решения и обязательства

Список решений и обязательств. Каждое обязательство в формате:
- **[Имя]** обязался: _что сделать_ | Срок: _дата_ | Цитата: _"..."_ [MM:SS]

Разделяй обязательства на **«новые»** (нет в `commitmentsBefore` по совпадению `who+what+deadline`) и **«продолжающиеся»** (совпадение есть). Обозначай словом «(продолжается)» в конце строки для продолжающихся.

Пометки на цитатах:
- `[approximate]` — если цитата не дословная (поле `approximate: true` в citations)
- `[speaker_check]` — если спикер требует верификации (имя в `speaker_check[]`)
- `[KK]` — если цитата на казахском (оставить дословно)
- `[KK/RU]` — для смешанных (code-switching) цитат

### Секция 2: OKR-покрытие

Краткая таблица:
- **Обсуждённые KR** (`status: 'discussed'`) — какие именно, ключевые выводы из анализа
- **Упомянутые KR** (`status: 'mentioned'`) — без подробного обсуждения
- **Слепые зоны** (`status: 'blind_zone'`) — выводи отдельной строкой `🔴 Слепые зоны: KR-X, KR-Y`

### Секция 3: Алерты и рекомендации (только если `alerts.length > 0`)

Перечисли алерты из `analysis.alerts` с короткой формулировкой действия. Если алертов нет — **секция отсутствует**, и `report_sections.length === 2`.

## Секция «📱 Для топа» (опциональный draft для трекера)

Если у топа есть **1+ обязательство с конкретным deadline на этой неделе**, сгенерируй `top_message_draft` — 3-5 строк текста для копирования трекером в WhatsApp/Telegram топу.

Формат:
- Тон: дружелюбный, конкретный, упоминает имя топа.
- Шаблон: «{Имя}, по итогам встречи: {ключевое решение}. Ты берёшь {commitment} к {deadline}.»

Если у топа **нет** обязательств с deadline на этой неделе — НЕ возвращай поле `top_message_draft` (оно опциональное).

## Формат ответа

Верни ТОЛЬКО валидный JSON (без markdown fences и без префикса/суффикса):

    {
      "report_sections": [
        {
          "title": "Название секции",
          "content": "Отформатированный текст секции в markdown"
        }
      ],
      "summary_line": "Одна строка-резюме встречи (до 200 символов)",
      "commitment_count": 3,
      "alert_count": 1,
      "top_message_draft": "Алия, по итогам встречи: ..."
    }

`top_message_draft` — опциональное поле; если у топа нет обязательств с deadline — пропусти его, а не возвращай пустую строку.

## Критически важно

- **Максимум 3 секции**. Если `alerts.length === 0` — 2 секции.
- Формат должен быть **сканируемым**: списки, **bold** для имён/метрик, короткие предложения.
- **Точность > Полнота**. Не добавляй то, чего нет в данных.
- Язык narrative — **русский**. Казахские цитаты — дословно с пометкой `[KK]`. Code-switching — целиком в оригинале с `[KK/RU]`.
- `summary_line` ≤ 200 символов.
- `section.content` ≤ 3500 символов.
- `commitment_count` = `extraction.commitments.length` (просто число, для UI бейджа).
- `alert_count` = `analysis.alerts.length`.
