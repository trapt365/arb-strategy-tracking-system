## Инструкции

Сгенерируй повестку из **максимум 3 пунктов**:

1. **Какие KR проверить** — KR из зоны ответственности менеджера, которые:
   - Имеют срок в ближайшие 2 недели
   - Были в blind_zone на прошлой встрече
   - Показывают отставание по прогрессу

2. **Открытые обязательства** — невыполненные обязательства с прошлой встречи:
   - Кто, что обязался, какой был срок
   - Статус: выполнено / просрочено / в процессе (если известно)

3. **Слепые зоны и несоответствия** — области, требующие внимания:
   - KR не обсуждавшиеся 2+ недели
   - Расхождения между заявлениями и метриками (если F5 данные доступны)

}

## Критически важно

- Максимум 3 пункта повестки. Приоритизируй по важности.
- Каждый пункт должен быть **actionable** — конкретный вопрос или тема для обсуждения.
- Формулируй как направляющие вопросы для развития коучинговых навыков трекера.
- Не включай пункты, для которых нет данных. Лучше 2 качественных пункта, чем 3 с притянутыми.
