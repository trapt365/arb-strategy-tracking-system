# Дизайн-записка Story 8.5: два алгоритма входа онбординга — импорт готовой стратегии vs синтез

Дата: 2026-07-08 · Статус: **ЖДЁТ УТВЕРЖДЕНИЯ ТИМУРОМ** (AC1: кодирование только после утверждения)
Закрывает: FR102, W11 (`.xlsx` отвергается, хотя готовые стратегии живут в Excel).

## 1. Ключевое архитектурное решение

**Импорт — это альтернативный продюсер `F0FullExtraction`, а не параллельный пайплайн.**

Сегодня единственный вход: документы → `runF0FullDraft` (LLM) → `F0FullExtraction` → всё
остальное (черновик-саммари, инварианты 1–2, диалог дозаполнения 7.3/8.6, `/confirm` →
Sheets, карточка клиента). Импорт добавляет второй способ получить тот же
`F0FullExtraction` — без LLM. Весь даунстрим переиспользуется без единой правки:

```
mode=synthesis: файлы(md/txt/docx/pdf) → LLM → F0FullExtraction ┐
mode=import:    один .xlsx → src/f0-import.ts (0 LLM) ──────────┤→ общий путь:
                                                                 │  черновик → диалог
                                                                 │  пробелов → /confirm
                                                                 └→ _okr/_stakeholder_map/_hypotheses
```

Следствия: инварианты 1–2 применяются тем же кодом (`markBlockingKrIssues` /
`markHypothesesWithoutMetric`); неполные KR из Excel штатно попадают в 🔴 и в вопросы
дозаполнения; `/confirm` предупреждает-не-блокирует как везде. Нового состояния почти
нет — одно поле `mode` в сессии (optional в zod, совместимость 7.3).

## 2. Развилка UX (критерии выбора пути)

- `/newclient` (и `f0_new_yes`, и «новый клиент») → к приветствию collecting добавляются
  две кнопки: **«📥 Есть готовая стратегия (Excel)»** / **«🧠 Собрать из документов»**.
- Кнопка НЕ обязательна: выбор фиксируется и автоматически по первому файлу —
  пришёл `.xlsx` → `mode=import`; пришёл md/txt/docx/pdf → `mode=synthesis`
  (текущее поведение, ноль дополнительных кликов для Азизы).
- Смешивать пути в одной сессии нельзя: в `mode=import` второй файл/текстовый файл →
  подсказка «импорт работает по одному Excel; для пакета документов — /newclient и синтез»;
  в `mode=synthesis` пришедший `.xlsx` → подсказка с кнопкой «переключиться на импорт»
  (с guard-подтверждением, если уже накоплены файлы — паттерн W3).
- Импорт невозможен (см. §4) → бот честно говорит почему и предлагает синтез-путь
  (AC3: «или импорт невозможен»).

## 3. Маппинг xlsx → `F0FullExtraction`

Новый модуль `src/f0-import.ts` (чистые функции, xlsx `^0.18.5` уже в deps; паттерн
чтения ячеек — как в `scripts/extract-okr-from-xlsx.ts`: `cellToString` для
чисел/дат/процентов).

**Формат A — «наша» таблица** (xlsx-экспорт шаблона v2.0 / Geonline-стиль).
Детект: есть лист `_okr` (машинный) или лист с заголовками `kr_number|key_result|owner…`.
Маппинг прямой — обратный к `mapOkrRows`/`mapStakeholderRows`/`mapHypothesisRows`
(f0-sheets.ts): `key_result→formulation`, `current_status→base`, `target→target`,
`owner→owner`, `deadline→deadline`, `okr_group→objective.title` (группировка KR в
objectives по этой колонке); `_stakeholder_map`/лист участников → participants
(`full_name`, `role`, `department`, `telegram→contact`); `_hypotheses` (если есть) →
hypotheses (`synthesized:false`).

**Формат B — произвольная таблица.** Выбор листа: лист с максимумом совпавших
заголовков (порог ≥ 3 из: KR/ключевой результат/результат, база/текущее/current/с,
цель/target/до/план, ответственный/владелец/owner/кто, срок/дедлайн/deadline/когда,
направление/цель года/objective). Синонимы — таблица констант, регистронезависимо.
Objectives: по колонке направления, иначе один objective с названием листа.
Participants: уникальные имена из колонки «ответственный». Гипотезы: только если
распознан лист/блок гипотез; **синтез гипотез LLM-ом в импорт-пути НЕ делаем**
(«минимум LLM» = ноль вызовов; отсутствие гипотез — честный счётчик 0 в черновике,
Азиза добавит руками или прогонит синтез).

**Нераспознанное (инвариант 3, AC2):** не-замапленные колонки выбранного листа и
пропущенные листы → `extraction.unrecognized` строками вида
«лист „Стратегия": колонки „Вес", „Комментарий"» (cap как в черновике). Ничего не
выдумываем: пустая база → 🔴 → вопрос дозаполнения (уже с валидацией чисел 8.6).

**Отказ импорта:** нет листа с порогом ≥ 3 заголовков, 0 распознанных KR-строк,
файл >лимита/битый → `F0OnboardingError('import_unmappable', …)` с человеческим
текстом + предложение синтеза. Никакого «тихо получилось пусто».

`kr_type`: всегда `'metric'` (детект milestone-вех из Excel — выдумывание; строже =
лишний вопрос дозаполнения, а `/confirm` всё равно не блокирует).

## 4. Что сознательно НЕ делаем (и почему)

1. **LLM-ассист маппинга заголовков** — отложен: fallback при слабой эвристике уже
   есть (синтез-путь, где LLM и так работает). Если реальные файлы клиентов покажут,
   что эвристика мажет, добавим один дешёвый вызов «сопоставь заголовки» отдельной story.
2. **Импорт «структурированного текста» отдельным парсером** — не делаем: готовая
   стратегия в тексте отлично идёт через существующий синтез (регекс-парсер
   markdown-таблиц хрупок и дублирует LLM). «Текст» из FR102 покрыт синтез-путём.
3. **Несколько xlsx / xlsx+документы в одном пакете** — вне scope, по одному Excel.
4. **xls (старый бинарный), csv** — вне scope MVP (xlsx-пакет xls прочитает, но не
   обещаем; csv добавить тривиально позже, если попросят).

## 5. Объём изменений

- `src/f0-import.ts` (новый, ~250 LOC): workbook → `{extraction, mappingReport}` +
  синонимы заголовков; юнит-тесты на фикстурах (наш формат, произвольный, немаппируемый,
  смешанные типы ячеек) — фикстуры генерим xlsx-пакетом в тесте, бинарники в репо не кладём.
- `src/utils/f0-input.ts` / `f0-document.ts`: принять `.xlsx` (kind `'xlsx'`, buffer
  passthrough — текст не извлекаем).
- `src/bot.ts`: кнопки развилки + автодетект по расширению, `session.mode?:
  'import'|'synthesis'` (persist optional), ветка в `buildF0Draft` (mode=import →
  `f0-import` вместо `runF0FullDraft`; прогресс-тикер не нужен — импорт <1с, сразу итог),
  блок «Не распознано» черновика уже показывает `unrecognized`.
- `src/types.ts`: `mode` в `F0PersistedSessionSchema` (optional).
- Регресс-guardrail: синтез-путь не трогаем (smoke SAM + GeOnline, canary, весь vitest);
  `clientId==='geonline'` / `GEONLINE_F0_SHEET_ID` не затрагиваются вовсе.

Оценка: ~1 день dev + мини-ревью.

## 6. Открытые вопросы Тимуру (решить до кода)

1. **Образец реального файла.** Эвристика формата B откалибрована по Geonline-xlsx.
   Есть ли под рукой стратегия AIPLUS/другого клиента в Excel — скинуть для фикстуры?
   (Не блокер: формат A + синонимы покрывают разумный дефолт.)
2. **Текстовый импорт через синтез (§4.2) — ок?** Или нужен детерминированный парсер
   markdown/таблиц без LLM (я против: хрупко, дублирует синтез).
3. **Автодетект по расширению + кнопки (§2) — ок?** Альтернатива — только явные кнопки
   без автодетекта (лишний клик Азизе на каждый онбординг).
4. **Гипотезы в импорте не синтезируем (§3) — ок?** Альтернатива — после импорта
   предлагать кнопку «🧠 Досинтезировать гипотезы из этого же файла» (LLM по textified
   xlsx; +1 небольшая задача).
