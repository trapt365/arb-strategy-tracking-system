## 1. Суть проблемы (Issue Summary)

Эпик 9 закрыт по коду (9.1–9.7 done, 662 vitest). Живой прогон пилота вскрыл **стратегический дрифт**: продукт построил **систему управления клиентами** (тяжёлый онбординг: профиль топов + decision-maker + расширенный профиль + вопросник + 3 пути), а **ядро трекера — три отчёта — не работает целиком**:

1. **Ядро потеряно из фокуса.** Трекер сейчас обязан давать три вещи: (1) отчёт после встречи, (2) **недельный отчёт (главный результат)**, (3) трекер гипотез. Из них 9.7 (недельный) построен, но **не питается** (аудио встречи не принимается), а **трекер гипотез не существует вовсе**.
2. **Онбординг переусложнён (реверс 9.1).** Story 9.1 сделала профиль топов/decision-maker **обязательным** первым шагом. Пилот в прогоне: «неудобно… убрать decision-maker… язык сложный». Решение пилота: **прощающий вход** (что есть → извлечь → в таблицу, без допроса), тяжёлый профиль → `/advanced`.
3. **Аудио встречи не обрабатывается (D20/D21/D22).** Существующий клиент: загрузка `.m4a` встречи → бот не реагирует; «Недельный отчёт» → «Встреч за неделю не обработано». Спина к 9.7 разорвана.
4. **Баги свежих фич Эпика 9.** 9.2 grounding не флагнул смешение (профиль «Рога и копыта» → OKR собрались по ARB из документов — D2). 9.4 xlsx снова упал на реальном файле «ARB Solutions Стратегический трекер v1.1 (1).xlsx» (D1).
5. **Косметика (D8–D19, D23, D24).** Приветствие/названия, нет маркера активного клиента, длинные имена листов Sheets, `/start`-спам, залипший стейт, «Скоро» в help.

**Доказательства:** 24 дефекта с таймкодами (`defects-walkthrough-2026-07-10.md`); эталонные шаблоны трёх отчётов извлечены из geonline нед.8/12 (`report-templates-from-geonline.md`).

## 4. Детальные изменения артефактов

### 4.1 epics.md — добавить Epic 10 (после Epic 9)

**FRs (новые):**
- **FR112 [MVP]:** Бот принимает запись встречи (аудио/видео/файл `.m4a`/`.ogg`/`.mp4`, ссылку) для ВЫБРАННОГО существующего клиента → распознаёт как отчёт о встрече → Soniox-транскрипция → существующий F1-конвейер → результат персистится и питает недельный отчёт (9.7). Бот сам определяет, что входящий файл = встреча, и к какому клиенту относится (активный клиент).
- **FR113 [MVP]:** Онбординг нового клиента — **прощающий вход**: 🔑-минимум сужен до названия компании + сути бизнеса (топы/decision-maker/расширенный профиль БОЛЬШЕ не обязательны); «пришли стратегию — что есть» → извлечение OKR, недостающее показывается в таблице (не допрос в чате). Сбор топов/decision-maker/расширенного профиля вынесен за команду `/advanced` (код 9.1 сохранён, из основного потока убран).
- **FR114 [MVP]:** Grounding (9.2) флагует смешение: если имя компании из профиля расходится с названием/сущностями в загруженных документах → 🔴/вопрос трекеру, а не молчаливая сборка OKR по данным документа. Клиент↔данные не смешиваются.
- **FR115 [MVP]:** xlsx-импорт (9.4) принимает реальный файл «ARB Solutions Стратегический трекер v1.1 (1).xlsx» (словарь синонимов пополнен, файл — в тесты); при отказе — конкретный запасной путь.
- **FR116 [MVP]:** Трекер гипотез — третий тип отчёта: все гипотезы клиента, сгруппированные по департаментам/OKR, таблица обновлений статусов (прошлая неделя → эта) + новые гипотезы + сводка по статусам + выводы (шаблон geonline нед.12). Строится как «прошлая неделя + дельта» → бот хранит недельный снимок гипотез и номер недели.
- **FR117 [MVP]:** Golden-eval отчётов: недельный отчёт (9.7) и трекер гипотез (FR116) прогоняются на входах geonline нед.8/12 (транскрипты + снимок Excel) и сравниваются с эталонными отчётами практики — критерий приёмки КАЧЕСТВА структуры/полноты (тесты не судят читаемость).
- **FR118 [MVP]:** Косметика: маркер активного клиента виден постоянно (D23); приветствие/название бота обновлены (D8/D9); имена персональных листов Sheets сокращены до имени топа (D14/D24); устранены `/start`-дубли, залипший онбординг-стейт, повторяющийся футер, противоречивые маркеры готовности, «Скоро»-пункты в help (D15–D19).

**Ключевые guardrails (в AC каждой story):**
- 10.1: intake встречи только для активного/выбранного клиента (не смешивать клиентов); Soniox-адаптер существующий; F1-персист существующий; неделя без встреч → честное «встреч не обработано» (9.7 не трогаем).
- 10.2: реверс делать optional-полями (профиль остаётся в `ClientCard`, просто не обязателен); старые/прод-сессии с полным профилем совместимы; `/advanced` доступен и позже из карточки клиента.

- **Guardrail всего эпика (как Epic 9):** инварианты Epic 7 (1–4) без изменений; `clientId === 'geonline'` fallback и `GEONLINE_F0_SHEET_ID` не ломать; vitest + tsc + canary + golden зелёные после каждой story; боевая таблица Geonline не затрагивается.
