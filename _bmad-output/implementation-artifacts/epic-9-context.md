# Epic 9 Context: Фидбэк пилота v2 (WP-39 Ф2)

<!-- Generated from planning artifacts. Regenerate with compile-epic-context if planning docs change. -->

## Goal

Довести онбординг и ежедневный вход трекера до самостоятельной работы Азизы с реальными клиентами (якорь приёмки Ф2: Азиза сама онбордит 2 пилотных клиентов через бота). Эпик рождён из аудио-фидбэка пилота после живого прогона 2026-07-09 (онбординг «qubiq» прошёл end-to-end, но продукт не готов к самостоятельной работе): вводится обязательный «Профиль клиента» как единственный источник имён/названий для всех артефактов, очевидный стартовый flow действующего трекера, три способа завести стратегию (импорт Excel, вопросник с голосом, документы включая презентации) со сходимостью к одному артефакту, недельный отчёт по всем встречам клиента и ссылки вместо остаточных «полотен» текста.

## Stories

Порядок выполнения: **9.1 → 9.2 → 9.3 → 9.7 → 9.4 → 9.5 → 9.6** (9.7 передвинута вперёд решением Тимура — ценность для живого цикла Geonline, от дизайн-записки не зависит).

- Story 9.1: [CR-3] Профиль клиента — обязательный первый шаг онбординга
- Story 9.2: [CR-3+CR-4в] Grounding — профиль клиента как единственный источник имён и названий
- Story 9.3: [CR-2] Стартовый flow действующего трекера
- Story 9.7: [CR-7] Недельный отчёт трекера по клиенту
- Story 9.4: [CR-4а] Единая точка входа + презентации в пути «документы» + робастный xlsx
- Story 9.5: [CR-4б+CR-6] Вопросник с голосовыми ответами
- Story 9.6: [CR-5] Ссылки вместо полотен — остаток инвентаризации

## Requirements & Constraints

- **Профиль клиента двухуровневый:** критический минимум 🔑 (название компании, суть бизнеса, топы: имя/должность/полномочия/зона ответственности, decision maker) обязателен — без него онбординг стратегии не предлагается; расширенный профиль (история/партнёры, финансовая Точка А, оргструктура файлом, запрос и ожидания) опционален, предлагается после минимума и доступен позже из карточки клиента.
- **Grounding:** профиль — единственный источник имён/названий для ВСЕХ артефактов (заголовки Sheets, персональные листы «👤 {Имя}» по реальным топам, распределение KR по заявленным зонам; несовпадение → 🔴/вопрос, не молчаливая замена). LLM-обработка сырых документов и атрибуция реплик неразмеченных транскриптов — по списку участников профиля. Сессии без профиля (начатые до 9.1) работают как раньше: grounding в коде опционален, обязателен только для новых онбордингов.
- **Контент вопросов** (профиль — Часть A; вопросник стратегии) — из `docs/onboarding-questionnaire-v1.0.md` (создан из материалов Vault практики, утверждён).
- **Голос:** принимается в вопроснике, профиле и дозаполнении; распознавание — существующим Soniox-адаптером (ru+kk валидированы); каждый ответ подтверждается трекером («✅ Ок / ✏️ Править / 🎤 Заново») — в артефакты попадает только подтверждённый текст; лимит 5 мин; вне онбординга голос получает вежливый отказ. В подсказках диалогов явно: можно голосом 🎤 или документом 📎.
- **Компактность выдач:** ни одно пользовательское сообщение не длиннее ~15 строк без ссылки на артефакт; паттерн «саммари + счётчики + ссылка».
- **Недельный отчёт (9.7)** — внутренний инструмент трекера (НЕ путать с F3-lite для CEO из Epic 4): агрегат по всем обработанным встречам клиента за неделю (движение KR, коммитменты, паттерны) поверх персистированных результатов F1; пер-встречный `/report` не меняется; неделя без встреч → честное «встреч не обработано».
- **Инварианты Epic 7 (блокирующие, действуют без изменений):** (1) KR без числовой базы «с X до Y» и ответственного не выпускается — 🔴 + дозаполнение; (2) каждая гипотеза имеет метрику проверки; (3) не выдумывать: не извлёк → показать нераспознанное и спросить; (4) ничего не уходит клиенту без одобрения трекера.
- **Регресс-guardrail после КАЖДОЙ story:** `clientId === 'geonline'` fallback и `GEONLINE_F0_SHEET_ID` не ломать; canary + golden dataset + весь vitest + tsc зелёные; боевая таблица Geonline не затрагивается.
- **Вне scope:** голос в F1-трекинг-цикле, кодовый парсер структуры PPTX-слайдов, перенос оформления презентаций, диаризация голосовых ответов, QC-скоринг, веб-дашборд, смешивание путей онбординга в одной сессии.

## Technical Decisions

- **Расширение схем — только optional-полями zod** (паттерн 8.5/8.6): старые/активные прод-сессии восстанавливаются без миграции. Профиль → расширенная `ClientCard` (`src/f0-client-card.ts`, схема в `src/types.ts`) + новая фаза `profile` в персисте сессии; рестарт бота восстанавливает диалог с текущего вопроса.
- **Сходимость путей:** все три способа производят один артефакт (черновик F0-извлечения) и общий даунстрим `deliverF0Draft` → дозаполнение → `/confirm` → Sheets; после черновика ничего не дублируется. Автодетект по первому файлу сохранён (`.xlsx` → импорт, остальное → документы).
- **Презентации:** текст извлекается кодом (PDF — `pdf-parse`, уже в deps; PPTX — распаковка zip + текст из XML, `jszip` — единственная новая маленькая зависимость), дальше существующий LLM-синтез с grounding; для одиночной презентации готовой стратегии промпт ужесточён («перенеси, не досочиняй»).
- **xlsx-импорт (без LLM):** словарь синонимов пополнить по реальному файлу `ARB_Solutions_Стратегический_трекер_v1_1_1.xlsx` (отказ `import_unmappable` в прогоне пилота); файл — в тесты; при отказе — конкретный запасной путь.
- **Голос:** voice-сообщение TG → `getFile` (лимит TG 20 МБ) → `.oga` → существующий `src/adapters/soniox.ts` (uploadFile → poll → transcript; ogg/opus поддержан) → тот же обработчик, что печатный ответ. Новая транскрипция НЕ строится.
- **Вопросник** переиспользует механику 7.3 (persist, `/skip`, `/resume` — переживает рестарт) и валидацию чисел 8.6; ответственный за KR — кнопками из топов профиля (не свободный текст).
- **Конвенции проекта:** промпты только через `loadPrompt()` из `prompts/*.md`; Claude output только через `parseClaudeJSON(raw, Schema)`; Zod-схема на каждый chain step в `types.ts`; Sheets — чтение по имени заголовка; логирование pino с `{pipeline, step, clientId}`; `withRetry()` для API-вызовов; заголовки Telegram через `formatHeader()`; callback data `action:id`.
- **Формат stories — лёгкий (solo MVP):** 3 AC, ~5 задач, ~80 строк story doc, 1 проход ревью.

## UX & Interaction Patterns

- **Стартовый flow:** welcome 3–5 строк; клиенты из реестра — кнопками прямо в `/start` (не за промежуточным кликом); «Онбординг нового клиента» и «Что умеет бот» сохранены; пустой реестр → текущее поведение. После выбора клиента — явные действия (отчёт / статус / таблица); `/report` без URL — контекстная подсказка по активному клиенту, не общий `missing_arg`; произвольный текст после выбора клиента — контекстная подсказка. `/help` показывает то же меню, что `/start`; полная справка — за кнопкой «Что умеет бот».
- **Единый экран «Как заводим стратегию?»** после 🔑-минимума профиля, три кнопки: 📥 импорт Excel (без ИИ) / 💬 вопросник (можно голосом) / 📄 документы (протоколы, транскрипты, презентации). Заменяет полотно `F0_START_TEXT`. Презентации — честно в пути «документы», без «маски импорта».
- **Остатки полотен (9.6):** `/confirm`-предупреждения → счётчик «⚠️ N KR стоит дозаполнить» + ссылка на панель таблицы; кнопка «🔗 Подробнее» после F1-отчёта → реальная ссылка на таблицу/артефакт вместо заглушки «Скоро». `splitForTelegram` — страховка, не норма.
- Уже компактно, не трогать: доставка F1/F0 (8.3), карточка клиента, вопросы дозаполнения, статусы/ошибки, прогресс-тикер.

## Cross-Story Dependencies

- 9.1 — фундамент: профиль нужен 9.2 (grounding), 9.4 (экран способов после профиля), 9.5 (топы кнопками, голос в профиле), 9.7 (имена в отчёте).
- 9.2 grounding используется путями 9.4 (LLM-синтез документов/презентаций) и 9.7 (имена/названия в недельном отчёте).
- 9.3 даёт меню клиента, из которого вызывается недельный отчёт 9.7.
- 9.4 создаёт экран способов, на котором появляется кнопка «Вопросник» для 9.5.
- Анти-дубль CR-5: сокращение welcome — в 9.3; замена `F0_START_TEXT` — в 9.4; в 9.6 только `/confirm`-предупреждения, «🔗 Подробнее» и финальная контрольная сверка.
- Гейт дизайн-записки для 9.4–9.5 СНЯТ (записка утверждена Тимуром 2026-07-09); `docs/onboarding-questionnaire-v1.0.md` существует.
- 9.7 не зависит от 9.4–9.6; Epic 4 (F3-lite) позже сможет строиться на агрегате 9.7.
