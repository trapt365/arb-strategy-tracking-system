## Пользовательская история

Как **коуч практики (Азиза)**,
я хочу **загрузить полный пакет артефактов (протокол сессии, OKR-документ, нарратив) в md/docx/pdf**,
чтобы **бот извлёк банк гипотез и участников из любых форматов реальных стратегических документов**.

## Контекст и границы scope

Расширяет 7.1 по трём осям: (1) форматы docx/pdf → текст; (2) несколько файлов за сессию (аккумулируются, черновик собирается по команде/кнопке); (3) новые сущности — банк гипотез (ЕСЛИ-ТО-ПОТОМУ ЧТО + метрика проверки, инвариант 2) и участники (имя/роль/департамент/контакт).

**В scope:** doc→text слой (`mammoth` для docx, `pdf-parse` для pdf) + расширение `isSupportedF0Document`; аккумуляция файлов в `F0Session.documents[]`; кнопка/команда «собрать черновик»; расширенная схема извлечения `F0FullExtractionSchema` (objectives/krs + hypotheses + participants); промпт `f0-full-extraction.md`; валидатор инварианта 2 (гипотеза без метрики → 🔴); синтез гипотез из решений/инициатив с пометкой «требует подтверждения» (кейс SAM); рендер расширяется.

**Вне scope:** диалог дозаполнения и восстановление сессии (7.3); создание Sheets (7.4); карточка клиента (7.5); мультиклиентность (7.6). Извлечение по-прежнему одношаговое (один Claude-вызов над конкатенацией файлов); map-reduce по секциям — при необходимости отдельная история.

**Эталоны:** GeOnline BSC-версия (гипотезы уже в IF-THEN-BECAUSE), «трекер гипотез 10 неделя» (целевой формат банка), протокол SAM (гипотез НЕТ — синтез из решений D/инициатив, участники таблицей).

## Acceptance Criteria

1. **Given** пакет из нескольких файлов md/docx/pdf, **when** извлечение отработало, **then** черновик содержит панель OKR + банк гипотез (ЕСЛИ-ТО-ПОТОМУ ЧТО, метрика проверки) + участников с ролями; гипотеза без метрики помечена 🔴 (инвариант 2).
2. **Given** артефакт без явных гипотез (SAM), **when** извлечение отработало, **then** кандидаты в гипотезы синтезированы из решений/инициатив и помечены «требует подтверждения трекером».
3. **Given** документ с мусорными URL, RU/KZ code-switching и разноформатными датами, **when** извлечение отработало, **then** мусор отсечён до подачи в Claude, эмодзи-статусы сохранены как данные, формат дат — как в документе.

## Задачи

1. **doc→text** `src/utils/f0-document.ts`: `extractTextFromDocument(buf, fileName, mimeType)` — md/txt (существующий decode), docx (`mammoth.extractRawText`), pdf (`pdf-parse`); type-shim для pdf-parse; расширить `isSupportedF0Document`; unit-тесты (в т.ч. пустой/битый docx/pdf → F0OnboardingError).
2. **Схема** `types.ts`: `F0HypothesisDraftSchema` (statement/ifThenBecause/metric nullable/department nullable/synthesized bool), `F0ParticipantDraftSchema` (name/role nullable/department nullable/contact nullable), `F0FullExtractionSchema` (extends OKR + hypotheses[] + participants[]).
3. **Промпт** `prompts/f0-full-extraction.md` ({{documentText}}): OKR (как 7.1) + гипотезы (инвариант 2: метрика обязательна, нет в документе → null; синтез из решений → `synthesized:true`) + участники; few-shot IF-THEN + синтез-кейс; запись в CHANGELOG.
4. **Пайплайн + валидатор** `f0-onboarding.ts`: `runF0FullDraft()` (аккумулированный текст), `markHypothesesWithoutMetric()` (🔴 `no_metric`), рендер добавляет секции «Банк гипотез» и «Участники».
5. **Бот**: `F0Session.documents[]` аккумулирует принятые файлы; ack по каждому («📎 принят, N в пакете»); кнопка/команда «собрать черновик» → `runF0FullDraft` над конкатенацией; smoke на SAM-протоколе.
