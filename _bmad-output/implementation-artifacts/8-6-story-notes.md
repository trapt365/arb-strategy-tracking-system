## Сделано (код)

- **W5 (группировка по KR).** Факт-чек предпосылки: очередь `computeF0Gaps` уже с 7.3 шла
  по-KR (base → target → owner одного KR подряд) — формулировка W5 в proposal
  («все базы → все цели → все ответственные») не соответствовала коду. Реальное трение —
  каждый вопрос был отдельной длинной простынёй с повтором формулировки KR. Сделано:
  - заголовок сущности на ПЕРВОМ вопросе группы: `📍 KR O1.2 «…» — не хватает: база «с X»,
    цель «до Y», ответственный.` (`F0Gap.header`, optional);
  - короткие вопросы полей с ref («Цель «до Y» для KR O1.2?») — контекст не теряется
    при `/resume` посреди группы;
  - порядок «KR → гипотезы → контакты → расписание» и прогресс `(i/N)` не менялись (AC1).
- **W6 (мягкая числовая валидация).** `kr_base`/`kr_target` без единой цифры → один
  переспрос с подсказкой формата; следующий непустой ответ принимается как есть
  (легальны «нет данных», «70% → 90%» проходит сразу). Критерий = `hasNumeric`
  инварианта 1 (`/\d/`) — консистентно с тем, что потом покажет /confirm.
  `/skip` снимает ожидание повтора. Хелперы: `needsNumericAnswer`, `looksNumericAnswer`
  (`f0-fill.ts`).
- **Persist (AC3).** `F0Gap.header` и `F0PersistedSession.retryGapIndex` — `.optional()`
  в zod: сессии 7.3 (вкл. активную прод-сессию AIPLUS, 32 вопроса) валидны и
  восстанавливаются без миграции; переспрос переживает рестарт (закреплено тестом).
