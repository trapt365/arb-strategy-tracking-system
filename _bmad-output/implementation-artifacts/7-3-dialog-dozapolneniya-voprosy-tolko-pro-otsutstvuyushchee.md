## Контекст и границы scope

После `buildF0Draft` (7.2) бот вычисляет пробелы из черновика и ведёт диалог по одному пункту: спрашивает ТОЛЬКО отсутствующее, применяет ответы к черновику, позволяет пропустить (`/skip`) и подтвердить (`/confirm`). Инвариант 1 блокирует подтверждение, пока есть 🔴-KR. Состояние персистится на диск — переживает рестарт бота.

**В scope:** чистый модуль `src/f0-fill.ts` (`computeF0Gaps`, `applyF0Answer`); типы пробелов kr_base/kr_target/kr_owner/hypo_metric/participant_contact + финальный вопрос про расписание; фаза сессии collecting→filling→ready; персист сессии `data/.onboarding/session-{chatId}.json` (Zod-схема) + ленивое восстановление; команды `/skip`, `/confirm`, `/resume`; блокировка `/confirm` при незакрытых 🔴-KR (инвариант 1).

**Вне scope:** карточка клиента и чеклист готовности (7.5 — здесь только флаг ready + краткое сообщение); создание Google Sheets (7.4); мультиклиентность (7.6). Расписание собирается одним свободным ответом и хранится на сессии (структурирование — в карточку 7.5).

## Acceptance Criteria

1. **Given** черновик с 🔴-KR, гипотезами без метрики и участниками без контактов, **when** трекер продолжает онбординг, **then** бот задаёт вопросы ТОЛЬКО про отсутствующее — по одному пункту, `/skip` пропускает и идёт дальше; ответ применяется к черновику.
2. **Given** трекер шлёт `/confirm`, **when** остались незакрытые 🔴-KR (нет базы/ответственного), **then** подтверждение заблокировано со списком пробелов (инвариант 1); когда блокеров нет — онбординг помечается ready.
3. **Given** сессия прервана (рестарт бота), **when** трекер возвращается (`/resume` или продолжает диалог), **then** состояние (черновик, очередь пробелов, позиция, ответы) восстановлено с диска с места остановки.

## Задачи

1. **Чистый модуль** `src/f0-fill.ts`: `F0Gap`/`F0GapKind`, `computeF0Gaps(extraction)` (из markBlockingKrIssues + markHypothesesWithoutMetric + участники без contact, + финальный schedule-пункт), `applyF0Answer(extraction, gap, answer)` (мутирует поле по локатору); unit-тесты.
2. **Схема персиста** `types.ts`: `F0PersistedSessionSchema` (phase, draftId, sourceNames, extraction, gaps, gapIndex, schedule); безопасная загрузка через Zod.
3. **Persist/restore** в `f0-onboarding.ts`: `persistF0Session(chatId, state)` (атомарно, session-{chatId}.json), `loadF0Session(chatId)` (Zod safeParse, null при отсутствии/битом).
4. **Диалог в боте**: после сборки — фаза filling, задаёт пробелы по одному; `message:text` применяет ответ и persist; `/skip`, `/confirm` (инвариант 1), `/resume`; ленивое восстановление сессии, если in-memory нет.
5. **Тесты**: computeF0Gaps/applyF0Answer (вкл. «только отсутствующее»), confirm-блокировка при 🔴-KR, round-trip persist/load; typecheck + весь vitest зелёные.
