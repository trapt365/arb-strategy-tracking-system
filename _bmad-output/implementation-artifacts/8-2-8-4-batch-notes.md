## Story 8.4: Стартовое меню, навигация, защита сессии

- W10: «Клиенты» → список из реестра (`loadRegistry`), клик `client:{id}` → карточка из
  `data/{id}/card.json` (`renderClientCardMessage`: участники/расписание/старт + мини-чеклист
  по полям карточки; полнота KR — в панели таблицы) — работает и для завершённых клиентов;
  без карточки — внятный fallback. Кнопка «✅ Работать с этим клиентом» (`client_use:{id}`)
  → выбор персистится в `data/clients/active-clients.json` (переживает рестарт — иначе
  /report молча уехал бы в geonline); `/report` без clientId идёт по активному клиенту
  (явный аргумент приоритетнее; протухший активный → warn + geonline-fallback).
  `/status` без сессии показывает карточку активного клиента.

  guard: при filling или collecting-с-файлами бот просит подтверждение
  (`f0_new_yes`/`f0_new_no`) с компанией и прогрессом «отвечено X из Y». `/cancel` —
  завершение сессии с подтверждением (`f0_cancel_yes`/`f0_cancel_no`), чистит память+диск.
- W9: `setMyCommands` + resume, skip, cancel.

  закреплено тестом).
- Тесты: +9 (меню, список/карточка/фолбэк, client_use+persist+/report-резолв,
