# Story 8.5: Два алгоритма входа онбординга — импорт готовой стратегии vs синтез

Status: in-progress  # 2026-07-09: дизайн-записка утверждена Тимуром, кодирование начато

## Решение (утверждённая дизайн-записка)

Полный текст: `_bmad-output/planning-artifacts/design-f0-import-vs-synthesis-2026-07-08.md`
(статус УТВЕРЖДЕНО ТИМУРОМ 2026-07-09). Ключевое:

- **Импорт — второй продюсер `F0FullExtraction`, не параллельный пайплайн.** Один `.xlsx`
  → `src/f0-import.ts` (0 LLM-вызовов) → тот же черновик → общий даунстрим (саммари,
  инварианты 1–3, диалог дозаполнения 7.3/8.6, `/confirm` → Sheets) без правок.
- **Развилка:** кнопки «📥 Есть готовая стратегия (Excel)» / «🧠 Собрать из документов»
  при `/newclient` + автодетект по расширению первого файла. Пути в одной сессии
  не смешиваются; отказ импорта → честный текст + предложение синтеза.
- **Маппинг:** формат A («наша» таблица, детект по листу `_okr`/знакомым заголовкам,
  обратный к `mapOkrRows`/`mapStakeholderRows`/`mapHypothesisRows`) + формат B
  (произвольная таблица: лист с максимумом совпавших заголовков-синонимов, порог ≥ 3,
  формулировка KR обязательна). Незамапленное → `extraction.unrecognized`. Пустое не
  выдумываем (инвариант 3). `kr_type` всегда `'metric'`.

## Ответы Тимура на открытые вопросы записки (2026-07-09)

1. Фикстуры — **только Geonline** (реальных файлов других клиентов не используем).
2. Стратегия текстом — через существующий синтез-путь: **ок**.
3. Автодетект по расширению + необязательные кнопки: **ок**.
4. Гипотезы при импорте не синтезируем, но **добавить кнопку** «🧠 Досинтезировать
   гипотезы из этого же файла» после импорта (LLM по текстифицированному xlsx,
   существующий промпт `f0-full-extraction`, берём только hypotheses).
